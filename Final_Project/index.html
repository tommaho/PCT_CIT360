CIT360 Graph Assignment

Step 1

Study and complete the WDGraph class (a version is available on OneDrive). Understand how the Dijkstra’s single-source shortest path algorithm works.

Step 2

Use a Graph implementation to find the shortest path between two given locations. Locations are given via their geo coordinates. The information about the graph and the input points are given a file called “map.csv”

The format of the “map.csv” file is as follows (read carefully!):

V - An integer representing the total number of vertices
0, longitude, Latitude, height, location name
1, longitude, Latitude, height, location name
2, longitude, Latitude, height, location name
. .

E - An integer representing the number of edges
Vertex 1, Vertex 2, type (1 = one directional, 2 - bi-directional)
Vertex 1, Vertex 2, type
. .

N - an integer N representing number of cases that follows
origin vertex, destination vertex

Output

For each case in the input file, output the shortest path and total distance travelled. For each case:

  1. a) print the shortest path
  2. b) print the complete set of directions.
  3. c) print the total distance travelled

The formulas to calculate the distance between two GPS coordinate systems are readily available on the web. For instance you may use the ones sited in http://stackoverflow.com/questions/365826/calculate-distance-between-2-gps-coordinates or http://www.movable-type.co.uk/scripts/latlong.html

Upon completion, ZIP all your files (project) and submit. I should be able to run your programs from the folder that you submit. Remove any local packages and use local file references. Pay attention to the units of measurements do not mix up latitude with longitude. Verify your answers independently with the map.

Some technical requirements:

Name the input file “map.csv” Do not use any absolute file location for your input and output. Project must be self-contained and run as given without having to import any other source and support files. Write output to both standard output and file “directions.dat” Your code must be written by you and be documented well.

Extra credits available

The following features can earn you extra credits: Giving detailed directions such as “Walk North- North East for 200 feet” Giving more detailed directions such as “Turn Left, Walk North- North East for 200 feet” Creating 2D images and line connecting the way points
